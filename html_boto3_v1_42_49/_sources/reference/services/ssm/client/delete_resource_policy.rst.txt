delete_resource_policy
**********************



.. py:method:: SSM.Client.delete_resource_policy(**kwargs)

  

  Deletes a Systems Manager resource policy. A resource policy helps you to define the IAM entity (for example, an Amazon Web Services account) that can manage your Systems Manager resources. The following resources support Systems Manager resource policies.

   

  
  * ``OpsItemGroup`` - The resource policy for ``OpsItemGroup`` enables Amazon Web Services accounts to view and interact with OpsCenter operational work items (OpsItems).
   
  * ``Parameter`` - The resource policy is used to share a parameter with other accounts using Resource Access Manager (RAM). For more information about cross-account sharing of parameters, see `Working with shared parameters <https://docs.aws.amazon.com/systems-manager/latest/userguide/parameter-store-shared-parameters.html>`__ in the *Amazon Web Services Systems Manager User Guide*.
  

  

  See also: `AWS API Documentation <https://docs.aws.amazon.com/goto/WebAPI/ssm-2014-11-06/DeleteResourcePolicy>`_  


  **Request Syntax**
  ::

    response = client.delete_resource_policy(
        ResourceArn='string',
        PolicyId='string',
        PolicyHash='string'
    )
    
  :type ResourceArn: string
  :param ResourceArn: **[REQUIRED]** 

    Amazon Resource Name (ARN) of the resource to which the policies are attached.

    

  
  :type PolicyId: string
  :param PolicyId: **[REQUIRED]** 

    The policy ID.

    

  
  :type PolicyHash: string
  :param PolicyHash: **[REQUIRED]** 

    ID of the current policy version. The hash helps to prevent multiple calls from attempting to overwrite a policy.

**************
delete_service
**************



.. py:method:: VPCLattice.Client.delete_service(**kwargs)

  

  Deletes a service. A service can't be deleted if it's associated with a service network. If you delete a service, all resources related to the service, such as the resource policy, auth policy, listeners, listener rules, and access log subscriptions, are also deleted. For more information, see `Delete a service <https://docs.aws.amazon.com/vpc-lattice/latest/ug/services.html#delete-service>`__ in the *Amazon VPC Lattice User Guide*.

  

  See also: `AWS API Documentation <https://docs.aws.amazon.com/goto/WebAPI/vpc-lattice-2022-11-30/DeleteService>`_  


  **Request Syntax**
  ::

    response = client.delete_service(
        serviceIdentifier='string'
    )
    
  :type serviceIdentifier: string
  :param serviceIdentifier: **[REQUIRED]** 

    The ID or ARN of the service.

    

  
  
  :rtype: dict
  :returns: 
    
    **Response Syntax**

    
    ::

      {
          'id': 'string',
          'arn': 'string',
          'name': 'string',
          'status': 'ACTIVE'|'CREATE_IN_PROGRESS'|'DELETE_IN_PROGRESS'|'CREATE_FAILED'|'DELETE_FAILED'
      }
      
    **Response Structure**

    

    - *(dict) --* 
      

      - **id** *(string) --* 

        The ID of the service.

        
      

      - **arn** *(string) --* 

        The Amazon Resource Name (ARN) of the service.

        
      

      - **name** *(string) --* 

        The name of the service.

        
      

      - **status** *(string) --* 

        The status. You can retry the operation if the status is ``DELETE_FAILED``. However, if you retry it while the status is ``DELETE_IN_PROGRESS``, the status doesn't change.

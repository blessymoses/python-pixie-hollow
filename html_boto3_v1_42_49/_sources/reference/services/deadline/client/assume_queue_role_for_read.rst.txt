**************************
assume_queue_role_for_read
**************************



.. py:method:: DeadlineCloud.Client.assume_queue_role_for_read(**kwargs)

  

  Gets Amazon Web Services credentials from the queue role. The IAM permissions of the credentials are scoped down to have read-only access.

  

  See also: `AWS API Documentation <https://docs.aws.amazon.com/goto/WebAPI/deadline-2023-10-12/AssumeQueueRoleForRead>`_  


  **Request Syntax**
  ::

    response = client.assume_queue_role_for_read(
        farmId='string',
        queueId='string'
    )
    
  :type farmId: string
  :param farmId: **[REQUIRED]** 

    The farm ID of the farm containing the queue.

    

  
  :type queueId: string
  :param queueId: **[REQUIRED]** 

    The queue ID.

    

  
  
  :rtype: dict
  :returns: 
    
    **Response Syntax**

    
    ::

      {
          'credentials': {
              'accessKeyId': 'string',
              'secretAccessKey': 'string',
              'sessionToken': 'string',
              'expiration': datetime(2015, 1, 1)
          }
      }
      
    **Response Structure**

    

    - *(dict) --* 
      

      - **credentials** *(dict) --* 

        The credentials for the queue role.

          The IAM access key ID.

          The IAM secret access key.

          The IAM session token

          The expiration date and time of the IAM credentials.

************************
delete_route_server_peer
************************



.. py:method:: EC2.Client.delete_route_server_peer(**kwargs)

  

  Deletes the specified BGP peer from a route server.

   

  A route server peer is a session between a route server endpoint and the device deployed in Amazon Web Services (such as a firewall appliance or other network security function running on an EC2 instance). The device must meet these requirements:

   

  
  * Have an elastic network interface in the VPC
   
  * Support BGP (Border Gateway Protocol)
   
  * Can initiate BGP sessions
  

  

  See also: `AWS API Documentation <https://docs.aws.amazon.com/goto/WebAPI/ec2-2016-11-15/DeleteRouteServerPeer>`_  


  **Request Syntax**
  ::

    response = client.delete_route_server_peer(
        RouteServerPeerId='string',
        DryRun=True|False
    )
    
  :type RouteServerPeerId: string
  :param RouteServerPeerId: **[REQUIRED]** 

    The ID of the route server peer to delete.

    

  
  :type DryRun: boolean
  :param DryRun: 

    A check for whether you have the required permissions for the action without actually making the request and provides an error response. If you have the required permissions, the error response is ``DryRunOperation``. Otherwise, it is ``UnauthorizedOperation``.
